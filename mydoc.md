# نظام تخصيص الحساب في MyDoc

يشرح هذا الملف كيف يعمل نظام تخصيص واجهة التطبيق، المبني على ثلاثة أجزاء:

```
app/src/main/assets/re.ui.css   ← الألوان الافتراضية للثيم (قبل أي تخصيص شخصي)
app/src/main/assets/io.yml      ← إعدادات تشغيل عامة وتفعيل/تعطيل الميزات
util/ThemeManager.java          ← يقرأ الملفين أعلاه ويطبّقهما، ويدمجهما مع تخصيص كل مستخدم
```

## re.ui.css

ملف "شبيه بـ CSS" (وليس CSS حقيقياً — التطبيق أندرويد Native وليس ويب)، بصيغة
`--اسم-المتغيّر: قيمة؛` بسيطة يسهل تعديلها بدون فتح Android Studio:

```
--primary: #FF0033;
--accent: #3FB950;
--background: #0D1117;
```

`ThemeManager.getDefaultPrimaryColor()` يقرأ `--primary` منه ليكون اللون
الافتراضي لكل أزرار وعناصر التطبيق الأساسية (زر النشر العائم، شريط
التنقّل السفلي...) قبل أي تخصيص شخصي.

## io.yml

ملف إعدادات تشغيل بسيط (بصيغة `key: value`، سطر لكل إعداد، بدون تداخل
حتى لا نحتاج مكتبة YAML خارجية):

```
app_name: MyDoc
theme_default_swatch: red
feature_likes: true
feature_reposts: true
feature_follow: true
feature_red_key: true
customization_enabled: true
```

يُحمَّل عبر `ThemeManager.runIo(context)` في بداية كل Activity رئيسية
(هذا هو معنى "تشغيل io.yml")، ويُستخدم للتحقق من كون ميزة معيّنة مفعّلة
عبر `ThemeManager.isFeatureEnabled(context, "feature_follow")` مثلاً،
بحيث يمكن إيقاف أي ميزة اجتماعية مؤقتاً بتعديل سطر واحد في io.yml بدون
تعديل الكود نفسه.

## حاوية التخصيص في صفحة الحساب

من صفحة الحساب (`ProfileActivity`) ← قسم **الإعدادات** ← **حاوية التخصيص**،
يقدر المستخدم يختار لوناً أساسياً من مجموعة ألوان جاهزة (أحمر/أزرق/أخضر/
بنفسجي/برتقالي). عند الاختيار:

1. يُحفظ اللون كنص Hex في حقل `customPrimaryColor` داخل مستند المستخدم في
   Firestore (`users/{uid}`) — هذا تخصيص **شخصي خاص بحساب كل مستخدم**، لا
   يؤثر على باقي المستخدمين.
2. يُطبَّق فوراً على الشاشة الحالية (`ThemeManager.applyToActivity`)، وعلى
   كل الشاشات التالية بعد إعادة فتحها، لأن كل Activity تستدعي
   `ThemeManager.applyToActivity(this)` في `onCreate`.
3. إن لم يخصّص المستخدم لوناً أبداً، يبقى اللون الافتراضي هو ما في
   `re.ui.css`.

## لماذا هذا التصميم؟

- **re.ui.css / io.yml** يفصلان "الإعدادات القابلة للتعديل بسهولة" عن الكود
  البرمجي — أي شخص يقدر يفتح هذين الملفين النصيّين ويغيّر لوناً افتراضياً
  أو يعطّل ميزة، بدون فهم Java أو أندرويد.
- **customPrimaryColor في Firestore** هو التخصيص *لكل مستخدم على حدة*
  (وليس ملفاً مشتركاً)، لأن كل حساب مستقل ويُفترض أن يقدر يختار مظهره
  الخاص دون التأثير على غيره.
